# Requirements Documentation

This folder holds requirement-style documents that humans and agents may both
need before product or implementation changes.

Typical files:

- `SRS.md`
- `DFD.md`
- `use-cases.md`
- `non-functional-requirements.md`
- `SRDS.md`

Do not treat placeholders or imported templates as product truth. Mark uncertain
items as `unknown` until code, tests, existing docs, or a human confirms them.
